# 이슈 생성 스킬

사용자가 입력한 내용을 분석해 GitHub 이슈를 생성한다.

## 실행 순서

1. 사용자 입력에서 작업의 성격을 판단해 타입을 결정한다:
   - `feat` — 새로운 기능
   - `fix` — 버그 수정
   - `chore` — 빌드·설정·리팩토링·문서·테스트 등
2. 결정한 타입에 맞춰 `.github/ISSUE_TEMPLATE/{type}.yml` 을 읽어 필수 필드를 확인한다.
3. 사용자 입력을 토대로 각 필드를 채워 이슈 본문 초안을 작성한다. 필드는 `### {label}` 형식의 마크다운 섹션으로 변환한다.
4. 제목은 `[{type}] {요약 (50자 이내)}` 형식으로 작성한다.
5. 초안(제목 + 본문 + 라벨)을 사용자에게 보여주고 **반드시 확인받는다**. 승인 없이는 이슈를 생성하지 않는다.
6. 승인 후 `gh issue create` 로 이슈를 생성한다:
   ```bash
   gh issue create \
     --title "[type] 요약" \
     --label "type" \
     --body "$(cat <<'EOF'
   ### 섹션1
   ...
   EOF
   )"
   ```
7. 생성된 이슈 URL을 출력한다.

## 규칙

- 정보가 부족해 필수 필드를 채울 수 없으면 사용자에게 추가로 물어본 뒤 이슈를 생성한다.
- 템플릿의 `required: true` 필드는 빈 채로 두지 않는다.
- 제목·라벨은 템플릿의 `title`·`labels` 값을 그대로 따른다.
- 이슈를 생성하기 전에 반드시 사용자에게 초안을 보여주고 승인받는다.

## 실행

위 순서대로 바로 실행한다.
