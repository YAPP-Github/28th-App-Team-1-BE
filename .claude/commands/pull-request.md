# PR 스킬

현재 브랜치의 변경 사항을 분석하고 PR을 생성한다.

## 실행 순서

1. `git status`, `git log main..HEAD`, `git diff main...HEAD` 를 실행해 변경 내용과 커밋 목록을 파악한다.
2. 현재 브랜치명에서 이슈번호를 추출한다. (예: `feat/42-login` → `#42`)
3. `.github/PULL_REQUEST_TEMPLATE.md` 를 읽어 PR 본문 템플릿을 확인한 뒤, 해당 양식에 맞춰 초안을 채워 사용자에게 보여주고 확인받는다.
4. 확인 후 `gh pr create` 로 PR을 생성한다. base 브랜치는 `main` 으로 고정한다.
5. 생성된 PR URL을 출력한다.

## 규칙

- PR 제목은 커밋 컨벤션과 동일한 형식으로 작성한다: `<type>: <요약 (50자 이내)>`
- 브랜치명에서 이슈번호를 찾을 수 없으면 사용자에게 직접 물어본다.
- 원격 브랜치가 없으면 `git push -u origin <브랜치명>` 으로 먼저 푸시한다.
- PR을 생성하기 전에 반드시 사용자에게 초안을 보여주고 확인받는다.

## 실행

위 순서대로 바로 실행한다.
