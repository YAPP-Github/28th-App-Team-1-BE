# 커밋 스킬

변경 사항을 분석하고 lat.md 컨벤션에 맞는 커밋을 생성한다.

## 실행 순서

1. `git status`와 `git diff` 를 실행해 변경된 파일과 내용을 파악한다.
2. 변경의 성격을 분류한다:
   - `feat:` — 새로운 기능
   - `fix:` — 버그 수정
   - `refactor:` — 기능 변경 없는 코드 개선
   - `docs:` — 문서 작성·수정 (lat.md 포함)
   - `test:` — 테스트 추가·수정
   - `chore:` — 빌드·설정 등 기타
3. 커밋 메시지를 아래 형식으로 작성한다:
   ```
   <type>: <변경 요약 (한글 또는 영문, 50자 이내)>
   ```
   영향 범위가 넓거나 이유 설명이 필요하면 본문을 추가한다:
   ```
   <type>: <요약>

   - <상세 내용 1>
   - <상세 내용 2>
   ```
4. 관련 파일을 스테이징(`git add`)하고 커밋한다.
5. 커밋 결과를 출력한다.

## 규칙

- 하나의 커밋은 하나의 목적만 담는다. 성격이 다른 변경이 섞여 있으면 사용자에게 분리 여부를 먼저 물어본다.
- `.env`, 시크릿, 바이너리 파일은 절대 스테이징하지 않는다.
- `--no-verify` 등 훅 우회 옵션은 사용하지 않는다.
- 커밋 메시지 끝에 `Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>` 을 추가한다.

## 실행

위 순서대로 바로 실행한다. 사용자에게 메시지 초안을 먼저 보여주고 확인받은 뒤 커밋한다.
